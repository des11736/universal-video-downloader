# 视频列表封面、预览与标题设计

## 目标

优化 Web UI 的视频分析与下载任务列表：

1. 已分析视频的列表封面显示视频首帧；无法加载首帧时回退到平台缩略图或占位图。
2. 已完成下载的视频能在 Web UI 中预览，并在文件不可用或浏览器无法播放时给出可见错误。
3. 下载任务标题优先显示分析得到的视频名称；没有名称时显示下载文件名（不含扩展名），最后才显示原始 URL。

## 方案与数据流

`POST /api/info` 在保留格式列表的基础上返回可用于首帧显示的媒体 URL。前端视频卡片使用无控件、静音、内联播放的 `<video>` 元素作为封面，加载首帧后暂停；当该元素触发错误时，替换为已有的 `thumbnail`，再回退为占位图。

下载请求携带分析结果中的 `title`。后端把该标题写入任务状态；下载完成后，若任务没有标题，则从最终文件路径推导文件名（去扩展名）。任务查询、页面刷新和 WebSocket 更新均保留该字段，使列表刷新后仍显示同一标题。

预览继续通过同源 `/api/preview/{task_id}` 提供本地文件。接口仅允许已完成且实际存在的常见浏览器媒体文件，正确返回媒体类型和不存在错误；前端在打开弹窗后重置、设置来源并调用 `load()`，同时监听 `error` 事件，在弹窗中显示可操作的失败原因。

## 边界与回退

- 视频首帧来自媒体流，不下载完整视频；流地址过期、受访问策略限制或浏览器不支持时，使用平台缩略图或占位图。
- 视频号分析目前无法在不启动其抓包下载流程的情况下取得可播放媒体流，因此维持占位/缩略图回退，不改变现有下载流程。
- 标题与媒体 URL 均由现有可信的适配器分析结果产生；浏览器不接受用户拼接的媒体地址。
- 预览文件不存在或不是可预览格式时，API 返回明确的 404/415 响应，前端展示提示而不是静默失败。

## 测试

- 为信息接口增加媒体封面地址的返回测试。
- 为下载 API 与任务列表增加标题保存、文件名回退测试。
- 为预览接口增加存在文件、缺失文件、不可预览扩展名和范围请求的测试。
- 为静态前端添加可检查的断言，覆盖首帧封面元素、标题显示和预览错误处理。

## 非目标

- 不新增 FFmpeg、图像处理库或封面缓存目录。
- 不改变平台下载、解密或输出命名规则。
- 不改变用户现有的多链接输入改动。
