# 视频号页面监听与注入下载按钮设计

## 目标

用户双击 `scripts/start_uvd.bat` 后，UVD Web 服务立即启动一个长期运行的本地 MITM 代理。浏览器在系统代理设置为 `127.0.0.1:8888` 的前提下访问视频号页面时，代理自动注入下载按钮；点击该按钮后由浏览器直接下载并解密视频。

## 范围

- `uvd serve` 的生命周期负责创建证书、启动和停止唯一的 `WechatChannelsMitmProxy`。
- 启动 BAT 的状态输出说明视频号监听器与本地代理端口。
- 视频号页面中的注入按钮保留浏览器端 `fetch → 解密 → FileSaver` 下载方式。
- Web UI 不再提交视频号下载任务，改为返回明确提示，避免任务适配器再启动第二个 8888 代理。
- README 改为“启动 BAT、安装证书、设置代理、打开视频号页面、点击注入下载按钮”的操作流程。

## 非范围

- 不改变非视频号平台的 Web UI 下载流程。
- 不将页面按钮的浏览器下载改为后端任务下载。
- 不修改视频号网页的认证、登录或访问控制。

## 架构

`start_uvd.bat` 继续启动 `uvd serve`。FastAPI 启动时调用 `ensure_ca_cert()`，随后创建 `WechatChannelsMitmProxy(port=8888)` 并调用 `start()`；关闭时调用 `stop()`。代理只负责监听、拦截 HTML 和注入现有 `inject.js`，不等待 profile 或创建下载任务。

浏览器页面中的按钮保留现有直接下载逻辑。Web API 在收到视频号 URL 的普通下载请求时返回冲突错误，提示用户设置系统代理后到视频号页面点击注入按钮。这消除了服务生命周期代理与任务生命周期代理同时绑定 8888 的冲突。

## 失败处理

- 缺少 `mitmproxy`、证书生成失败或 8888 被占用时，Web 服务继续提供其他平台功能，并在控制台和 Web API 日志中给出具体错误。
- 根证书未安装时，监听器仍会启动；浏览器 HTTPS 校验会失败，README 说明必须先安装证书。
- 关闭 Web 服务时始终停止监听器并清理其子进程。

## 验证

- 单元测试验证 Web 服务启动会启动监听器，关闭会停止监听器。
- 单元测试验证视频号 URL 的 Web 下载请求被拒绝并返回页面按钮指引，其他 URL 仍照常创建任务。
- 启动脚本测试验证其说明包含自动视频号监听页面的提示。
- 手工验证：启动 BAT、设置代理、打开视频号页面，确认页面显示注入下载按钮。
